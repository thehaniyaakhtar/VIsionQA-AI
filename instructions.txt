Architecture
Upload Image -> FastAPI -> PIL loads image -> BLIP Processor -> BLIP Model -> Generate Response -> JSON Response
FastAPI: receives the request and loads the image  using PIL
Image is passed to a HuggingFace Processor, formats the image to models expectations
BLIP performs inferences/ generates caption or an answer
FastAPI returns generated text as JSON

During training, the network gradually discovers visual features that help 
distinguish one object from another. 
Early layers may learn to detect simple edges, while deeper 
layers recognize increasingly complex structures such as

Vision Transformers
Transformers for images, dividing a picture into smaller patches
each patch is treated as a word in a sentence
patches -> numerical representations, embeddings

CLIP and BLIP
multimodal models: work on both images and texts
CLIP: determines whether an image and sentence are semantically related by 
mapping both into the same embedding space
learns a shared representation for images and text

BLIP: incorporates a language generation component
does not measure similarity, generates new text
enables it to produce captions describing an image/ answer questions 

Input format processing
grayscala image: 0-255
r, g, b
for an image with dimensions: 800x600 = 480000 pixles
each pixel: 3 values
48000 x 3 numbers as input for a single image

main.py: main entry point
when someone starts a server, FastAPI runs main.py
